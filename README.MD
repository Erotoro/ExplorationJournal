# 📖 Exploration Journal (Журнал Исследователя)

![Version](https://img.shields.io/badge/version-1.0.0-blue.svg)
![Platform](https://img.shields.io/badge/platform-Paper%20%7C%20Spigot%20%7C%20Folia-green.svg)
![Java](https://img.shields.io/badge/java-21%2B-orange.svg)

**Exploration Journal** — это продвинутый плагин для серверов Minecraft, который превращает исследование мира в захватывающее приключение. Игроки могут записывать открытые биомы, найденные структуры и уникальные открытия, повышая уровень своего журнала и открывая новые возможности.

---

## ✨ Основные возможности

*   🧭 **Система уровней журнала (I-IV):** Разблокируйте новые функции по мере прогресса.
*   🏰 **Исследование структур:** Плагин учитывает время, проведенное в структуре, для её «засчета».
*   🌍 **Интеграция с BlueMap:** Отображение открытий игроков на 3D-карте сервера.
*   💬 **DiscordSRV:** Логирование важных открытий прямо в ваш Discord-канал.
*   📜 **Экспорт в книгу:** Возможность создать физическую книгу со списком всех ваших достижений.
*   📊 **PlaceholderAPI:** Вывод статистики игрока в скорборды или меню.
*   🚀 **Поддержка Folia:** Полная совместимость с многопоточными ядрами.

## 🛠 Крафт журналов

Журналы создаются на верстаке. Каждый следующий уровень требует предыдущий.

### [Уровень I] Походный журнал
*Базовая запись биомов и структур.*
```text
[ Перо  ] [ Чернила ] [ Перо  ]
[ Бумага ] [ Книга   ] [ Бумага ]
[ Бумага ] [ Кожа    ] [ Бумага ]
```

### [Уровень II] Журнал Картографа
*Добавляет команду `/journal map` для просмотра координат открытых биомов.*
```text
[ Компас ] [ Золото  ] [ Компас ]
[ Золото ] [ Журнал I] [ Золото ]
[ Бумага ] [ Золото  ] [ Бумага ]
```

### [Уровень III] Журнал Исследователя
*Добавляет команду `/journal export` для создания книги со списком открытий.*
```text
[ Изумруд] [ Зелье   ] [ Изумруд]
[Журнал II] [ Тотем   ] [Журнал II]
[ Изумруд] [ Эндер-жем] [ Изумруд]
```

### [Уровень IV] Журнал Легенды
*Добавляет серверный топ `/journal top` и визуальные эффекты при открытии.*
```text
[ Незерит] [ Кристалл ] [ Незерит]
[ Незерит] [Журнал III] [ Незерит]
[ Незерит] [ Маяк     ] [ Незерит]
```

## 🎮 Команды и Права

### Игровые команды (`/journal` или `/ej`)
| Команда | Описание | Требуемый Tier | Право |
| :--- | :--- | :---: | :--- |
| `/journal` | Открыть журнал (интерфейс книги) | I | `explorationjournal.use` |
| `/journal biomes` | Список открытых биомов | I | `explorationjournal.use` |
| `/journal structures` | Список открытых структур | I | `explorationjournal.use` |
| `/journal discoveries` | Список уникальных открытий | I | `explorationjournal.use` |
| `/journal stats [player]` | Просмотр статистики | I | `explorationjournal.use` |
| `/journal map` | Координаты ваших биомов | II | `explorationjournal.use` |
| `/journal export` | Создать книгу-отчёт | III | `explorationjournal.export` |
| `/journal top [cat]` | Топ игроков (biomes/structures/disc) | IV | `explorationjournal.top` |
| `/journal give <player>` | Передать копию своего журнала | I | `explorationjournal.give` |

### Админ-команды (`/journaladmin` или `/eja`)
*Все команды требуют право `explorationjournal.admin`*
*   `/journaladmin reset <player>` — Полный сброс данных игрока.
*   `/journaladmin give <player> <1-4>` — Выдать журнал определенного уровня.
*   `/journaladmin reload` — Перезагрузка конфигураций плагина.

## 📥 Установка

1.  Скачайте последнюю версию плагина со страницы [Releases](https://github.com/ВашПрофиль/ExplorationJournal/releases).
2.  Поместите `.jar` файл в папку `plugins` вашего сервера.
3.  Запустите сервер для генерации файлов конфигурации.
4.  *(Опционально)* Настройте `config.yml` и `messages.yml` под свои нужды.
5.  Перезагрузите сервер или используйте `/journaladmin reload`.

## 🧩 Интеграции

*   **PlaceholderAPI:**
    *   `%explorationjournal_level%` — Уровень журнала игрока.
    *   `%explorationjournal_biomes_found%` — Кол-во найденных биомов.
    *   `%explorationjournal_structures_found%` — Кол-во найденных структур.
*   **BlueMap:** Автоматически создает маркеры в местах, где игроки совершают уникальные открытия.
*   **DiscordSRV:** Отправляет сообщения в указанный канал при первом открытии структуры или биома на сервере.
